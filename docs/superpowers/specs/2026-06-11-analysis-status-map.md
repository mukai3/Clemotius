# Clemoutis 解析状況マップ

作成日: 2026-06-11
目的: オリジナル（かざぐるマウス）の解析で「何が確定し、何が未確定か」を一覧化し、
今後どこを調べれば実装が進むかを示す。全関数デコンパイルはコスト不見合いのため行わない方針。

## 解析手段の整理

| 手段 | できること | 環境 |
|---|---|---|
| 静的解析（PEインポート/リソース/ヘルプ） | 構造・API・UIラベル・設定形式・選択肢名 | 実施済み（このリポジトリで完結） |
| 逆アセンブル（Ghidra/IDA） | コード値↔意味、アルゴリズム詳細 | 高コスト。狙い撃ちのみ想定 |
| 動的解析（VMで実機観測） | 各操作→ini値/送出メッセージの対応 | **ユーザー実機作業が必要**（GUI操作＋観測） |

## 確定済み（静的解析で判明）

- 構成: `Kazaguru.exe`(本体), `Kazasub.dll`(設定UI=17ダイアログ), `Kazahook*.dll`(フック, shared セクション=DLLインジェクション), `Kazawow64.exe`(32bitブリッジ)
- アクション機構: キー送信(SendInput/keybd_event) ＋ ウィンドウメッセージ(SendMessage/PostMessage/SendNotifyMessage) のハイブリッド。AttachThreadInput+SetForegroundWindow で確実配送
- UI要素判定: OLEACC(MSAA) 使用
- 設定ファイル: UTF-16LE INI（BOM無）。`[Settings]`/`[Gestures]`/`[MouseAssignment]`
- ジェスチャー値の2形式: 整数=組み込みコマンドID / `#1,VK,Shift,Ctrl,Alt,Win`=キー送信
- 設定画面タブ: オプション/ホイール/ボタン/拡張スクロール/マウスジェスチャー/シェル操作/ウィンドウ/詳細
- 拡張スクロール修飾キー: 6種（Shift/Ctrl/Ctrl+Shift/Alt/Shift+Alt/Ctrl+Alt）
- ホイールタブ: 垂直/水平スクロールバー上の挙動が各ドロップダウン
- アプリ種別カテゴリ: IE系/Mozilla系/Opera/Safari系/Chromium系/エクスプローラ/ウィンドウ（Chromiumは独立カテゴリ）
- コマンドカタログの一端（メニューリソースより）: コピー/取得系（選択テキスト・ブラウザURL/タイトル・リンク先URL/テキスト・エクスプローラのフォルダ/選択ファイル・ウィンドウの実行ファイル/タイトル 等）

## 未確定（動的解析または逆アセンブル待ち）

| 項目 | 取り方 | どの実装に効くか | 優先度 |
|---|---|---|---|
| `ScrollEx*` コード `53`/`55`/`58` の意味 | 動的: オプションで各挙動を選び ini の値変化を観測 | `onVerticalScrollbar`/`onHorizontalScrollbar`/Alt系の `code:NN` を名前付き挙動へ昇格。使用感の忠実再現 | 高 |
| 組み込みコマンドID（`L=1`,`R=2`,`LR=101`,`DL=112`,`U=5`,`UD=3`…）の意味 | 動的: 各コマンドを割当→実行して送出を観測 / 逆アセンブル: コマンドテーブル | ジェスチャーのアクション拡張。WM_APPCOMMAND か内部処理かの確定 | 中 |
| 各アクションの実送出機構 | 動的: Spy++/API Monitor | 挙動一致の受け入れ基準 | 中 |
| 工場出荷時デフォルト割り当て | 動的: ini削除→初回起動で再生成 | 新規ユーザー向け（自分用のため価値低） | 低 |
| 標準スクロールバーの厳密判定 | 逆アセンブル/MSAA実装 | ブラウザ等カスタム描画スクロールバーの検出 | 中 |

## 方針

- 重い一括解析（全関数デコンパイル）は行わない。言語/構造が別物（C++ネイティブ→C#）で移植に直結せず、コスト過大・法的にもグレーが濃い。
- 必要時に **狙い撃ち**で解析。特に「高」優先の `ScrollEx*` コードは、ユーザーが実機でオリジナルを操作できるタイミングで動的解析するのが最短。
- それまでは未確定コードを `code:NN` として保持（素通し）し、使用感を壊さない。

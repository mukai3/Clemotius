# 設定画面：バイナリ実物 vs Clemoutis 実装 比較

作成日: 2026-06-11
出典: `Kazasub.dll` の RT_DIALOG リソース（17ダイアログ）を抽出して解析

## オリジナルのダイアログ対応

| ダイアログ | 役割（タブ/モーダル） |
|---|---|
| 105 | **ホイール** タブ |
| 107 | **拡張スクロール** タブ |
| 108 | **マウスジェスチャー** タブ |
| 109 | **シェル操作** タブ（タスクバー/エクスプローラ拡張） |
| 112 | **ウィンドウ** タブ（タイトルバー/ウィンドウ/コントロール拡張） |
| 124 | **ボタン** タブ（中央/拡張1/拡張2 ボタン割当） |
| 113 | **詳細** タブ（122コントロールの巨大ページ。各機能の詳細＋アプリ/トレイ/性能） |
| 102 | バージョン情報 / 111 全ウィンドウ閉じ確認 / 119 ショートカットキー入力 / 122 外部ツール / 123 マウス入力割当 / 127〜129 セットアップ |

Clemoutis は v1 スコープに合わせ **4タブ**（マウスジェスチャー / 拡張スクロール / ホイール / 一般・詳細）に集約。対象外: シェル操作・ウィンドウ・ボタン。

## タブ別の相違

### 拡張スクロール（オリジナル 107 vs Clemoutis）

| 項目 | オリジナル | Clemoutis | 判定 |
|---|---|---|---|
| 修飾キー | Shift / Ctrl / Ctrl+Shift / Alt / Shift+Alt / Ctrl+Alt の**6種** | 同6種 | ✅ 一致 |
| 各修飾キーの挙動 | ドロップダウンで挙動を選択 | コンボで none/horizontal/未確定コード | ✅ 同形（挙動の選択肢は未デコード分が少ない） |
| 有効にするウィンドウ | **すべて / 標準スクロールバーを持つ窓 / ブラウザ / 単独スクロールバーを持つ窓 / テキストエディタ / その他** の6チェック | **無し**（スクロールは全体一律） | ⚠️ 相違 |

→ 重要: オリジナルは「**標準スクロールバー（非クライアントの WS_HSCROLL）**」と「**単独スクロールバー（独立 ScrollBar コントロール）**」を明確に区別している。Clemoutis の `ScrollBarDetector` は後者（単独）しか検出できず、これは既知の制約どおり。前者の検出には MSAA が要る（設計の D 課題）。

### ホイール（オリジナル 105 vs Clemoutis）

| 項目 | オリジナル | Clemoutis | 判定 |
|---|---|---|---|
| カーソル下を常にスクロール | チェック | 無し | ⚠️ 対象外（Win10標準） |
| カーソル下を前面化 | チェック | 無し | ⚠️ 対象外 |
| カーソル位置にフォーカス | チェック | 無し | ⚠️ 対象外 |
| 垂直スクロールバー上でホイール回転 | **挙動ドロップダウン** | 無し | ❌ 未対応 |
| 水平スクロールバー上でホイール回転 | **挙動ドロップダウン** | on/off チェック1個に簡略化 | ⚠️ 簡略化 |
| スクロールの加速（感度/加速度） | チェック+2値 | 無し（config値は保持） | ⚠️ 対象外 |

→ オリジナルは「垂直/水平スクロールバー上」の双方に**挙動の選択**を持つ。Clemoutis は水平のみ・on/off に簡略化。

### マウスジェスチャー（オリジナル 108 vs Clemoutis）

| 項目 | オリジナル | Clemoutis | 判定 |
|---|---|---|---|
| 割当方式 | ストローク一覧から選び「**割り当てるコマンド**」をカテゴリ付きカタログから選択（＋キー送信型） | ストローク自由入力 ＋ アクション3種（キー送信 / コマンド(WM_APPCOMMAND) / 閉じる） | ⚠️ モデル差 |
| 有効化対象 | **アプリ種別カテゴリ**：IE系 / Mozilla系 / Opera / Safari系 / Chromium系 / エクスプローラ / ウィンドウ（各「詳細」付き） | **プロセス名プロファイル**（近代化） | ⚠️ 設計上の意図的相違 |
| 左ボタン長押しの用途 | あり（ヘルプ記載） | 無し | ❌ 未対応 |

→ 最大の概念差。オリジナルは「ストローク→**組み込みコマンドカタログ**（戻る/進む/閉じる/タブ操作/ズーム/スクロール/音量/ウィンドウ操作…多数）＋キー送信」。Clemoutis は実用サブセット（キー送信 / WM_APPCOMMAND / WM_CLOSE）に限定。組み込みコマンドID の意味は D.1 で確定予定。

### 一般・詳細（オリジナル 113「詳細」+アプリ群 vs Clemoutis）

Clemoutis がカバー: トレイ表示・バルーン、判定範囲（=1ストロークの距離）・入力開始タイムアウト・長押し待ち時間・軌跡描画・線幅・有効/無効色。

オリジナル 113 にあって Clemoutis UI に**未露出**（config値は一部保持）:
- 描画方法（drawingType）／ジェスチャー中左クリックで連続実行（rapidFire）／ツールチップ表示の待ち時間
- ホイール詳細（無効窓へのホイール・蓄積結合・高解像度・実行間隔・自動補正・コントロール適用）
- アプリ詳細（アイコン各クリック動作・二重起動時動作・ダイアログ位置記憶・性能/フック制限緩和）

→ いずれも対象外機能か、config に値だけ保持して UI 未露出。

## ラベル文言の差（軽微）

オリジナルの呼称に寄せると一貫性が上がる箇所:
- 「判定範囲(px)」→ オリジナルは「**1 ストロークの距離**」
- 「タイムアウト(ms)」→「**入力開始のタイムアウト**」
- 「長押し判定(ms)」→「**長押しの待ち時間**」

## 結論

- **構造の骨格は一致**（タブ＝プロパティシート、拡張スクロールの修飾6種、軌跡/判定の詳細項目）。
- **意図的な相違**（スコープ/近代化）: アプリ種別カテゴリ→プロセスプロファイル、組み込みコマンドカタログ→3アクション、多数の対象外機能。
- **要検討の相違**:
  1. 拡張スクロールの「有効ウィンドウ」区分が無い（特に標準/単独スクロールバーの区別＝検出範囲に直結）
  2. ホイールの「垂直スクロールバー上」挙動が無く、水平も on/off に簡略化
  3. ラベル文言をオリジナル呼称へ寄せられる
